# FastDoor Bot

Telegram-бот для установщиков дверной компании. Бот позволяет установщикам авторизоваться по номеру телефона и получать уведомления о новых заказах от бэкенда.

---

## Стек

- **Python 3.10+**
- **aiogram 3** — Telegram Bot framework
- **FastAPI** — внутренний HTTP-сервер для приёма заказов от бэкенда
- **httpx** — HTTP-клиент для запросов к бэкенду
- **uvicorn** — ASGI-сервер
- **python-dotenv** — переменные окружения

---

## Архитектура

```
Бэкенд (fast-door.ru)
    │
    ├── GET  /api/installer/phone/{phone}   — проверка номера установщика
    ├── PUT  /api/installer                 — сохранение TgId
    ├── POST /api/sms/sendVerificationMessage — отправка SMS-кода
    └── POST /api/login                     — авторизация сервиса
    
Telegram Bot (polling)
    └── FSM авторизация: /start → номер → SMS-код → авторизован

FastAPI (0.0.0.0:8081)
    └── POST /send-message — бэкенд отправляет сообщения установщикам
```

git clone <repo>

pip install aiogram httpx fastapi uvicorn python-dotenv aiohttp logging

## Переменные окружения

Создать файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота

ENV=dev
BACKEND_DEV_URL=http://localhost:8080
BACKEND_PROD_URL=https://fast-door.ru

SERVICES_LOGIN=services
SERVICES_PASSWORD=services
```

| Переменная | Описание |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `ENV` | Режим запуска: `dev` или `prod` |
| `BACKEND_DEV_URL` | URL бэкенда для разработки |
| `BACKEND_PROD_URL` | URL бэкенда для продакшна |
| `SERVICES_LOGIN` | Логин сервисной учётной записи на бэке |
| `SERVICES_PASSWORD` | Пароль сервисной учётной записи на бэке |

## Флоу авторизации

1. Установщик пишет `/start`
2. Бот проверяет локальный `authorized_users.json` — если уже авторизован, сообщает об этом
3. Если не авторизован — просит отправить номер телефона
4. Бот проверяет номер через `GET /api/installer/phone/{phone}`
5. Если номер найден — отправляет SMS-код через `POST /api/sms/sendVerificationMessage`
6. Установщик вводит код — 3 попытки, после чего блокировка на 15 минут
7. При верном коде — `PUT /api/installer` сохраняет `TgId` на бэке, `tg_id` записывается в `authorized_users.json`

## API бота: POST /send-message

Бэкенд использует этот эндпоинт чтобы отправить сообщение конкретному установщику.

**URL:** `http://localhost:8081/send-message`

**Метод:** `POST`

**Тело запроса:**
```json
{
  "TgId": "1035363841",
  "message": "Новый заказ #123 — ул. Ленина 5, кв. 10"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `TgId` | string | Telegram ID установщика |
| `message` | string | Текст сообщения (обрезается до 4096 символов) |

**Успешный ответ:**
```json
{
  "status": "sent",
  "TgId": "1035363841"
}
```

**Ошибки:**

| Код | Причина |
|---|---|
| `500` | Пользователь заблокировал бота |
| `500` | Неверный TgId (chat not found) |
| `500` | TgId не передан или равен null |

**Swagger UI:** `http://localhost:8081/docs`

## Хранилище авторизованных пользователей

Файл `authorized_users.json` создаётся автоматически рядом с `bot.py`:

```json
{
  "tg_ids": [
    1035363841,
    987654321
  ]
}
```

Файл сохраняется при каждой успешной авторизации и читается при старте бота. Это позволяет боту помнить авторизованных пользователей после перезапуска.

## Логирование

Все события логируются в консоль с временными метками:

| Префикс | Уровень | Событие |
|---|---|---|
| `[LOGIN]` | INFO | Статус логина на бэке |
| `[INSTALLER]` | INFO | Запрос установщика по телефону |
| `[SMS]` | INFO | Отправка SMS-кода |
| `[PUT]` | INFO | Обновление TgId на бэке |
| `[START]` | INFO | Пользователь написал /start |
| `[AUTH]` | INFO | Успешная авторизация |
| `[SENT]` | INFO | Сообщение отправлено |
| `[BLOCK]` | WARNING | Пользователь заблокирован |
| `[CODE]` | WARNING | Неверный код |
| `[FAIL]` | ERROR | Ошибка отправки сообщения |
| `[FATAL]` | CRITICAL | Бот не смог стартовать |
